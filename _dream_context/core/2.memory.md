## Technical Decisions

<!-- Long-lived architectural decisions that are referenced repeatedly. Granular implementation decisions live in knowledge/technical-decisions-archive.md. Decisions that grow into reusable patterns get their own knowledge file. -->

- **`setup` is the single onboarding front door (2026-06-06)**: `init` alone scaffolds `_dream_context/` but installs no `.claude/` skills/agents/hooks — a half-install. All entry points (interactive menu, README, standalone `init`) now funnel to `setup`. `init` self-funnels via a finish-offer or loud warning, gated on SETUP_INTERNAL_ENV, TTY, and an `.every()`-over-platforms `integrationPresent` check. Source: src/cli/commands/setup.ts, src/cli/commands/init.ts; commits ecfe365, 486d418.
- **Recall/eval tests must exclude gitignored local captures (2026-06-06)**: `buildCorpus()` reads the live working tree, which on dogfooding machines contains `state/.session-digests/` and `.sleep.json` bookmarks (gitignored, never shipped). Tests asserting a capture-free baseline must filter `digest#`/`bookmark#` slugs or they pass in CI and fail locally. Source: src/lib/recall.ts; commit f9624c4.
- **License: MIT to Apache-2.0 (2026-06-06)**: Rationale and trademark stance (forks must use distinct names for 'dreamcontext'/'meansoft') documented in `knowledge/licensing-policy.md`. NOTICE + TRADEMARK.md + CONTRIBUTING.md (DCO) added. Source: commit a020584; knowledge:licensing-policy.
- **CSS Motion Path for SVG flow animations (2026-06-06)**: Animate `offset-distance` on a small dot element along an `offset-path` SVG path, NOT `stroke-dashoffset` on the full path. `stroke-dashoffset` re-rasterizes the entire stroke on the main thread every frame (visible stutter at >10 dots). Motion Path is compositor-class — only a tiny region is dirty per frame; measured worst-frame 9-11ms, 0 jank at ~27 dots. Glow = radial-gradient fill on the dot (rasterized once), not a per-frame drop-shadow filter. Source: dashboard/src/components/about/FlowDiagram.css.
- **`overflow-x: clip` vs `overflow-x: hidden` for sticky descendants (2026-06-06)**: `overflow-x: hidden` establishes a scroll container on that axis (the other axis computes to `auto`), which causes `position: sticky` descendants to resolve to that element as their scroller rather than the real page scroller — breaking the sticky behavior entirely. Use `overflow-x: clip` to clip horizontally without establishing a scroll container. Source: dashboard/src/components/about/AboutPage.css; blocking issue discovered during sticky scrub section implementation.
- **bundleDir skill-pack install mechanism (2026-06-05)**: CatalogStandalone entries with `bundleDir:true` copy the full directory via cpSync rather than a single SKILL.md, preserving binaries and exec bits. CJS subtrees under the ESM host must include a `package.json {"type":"commonjs"}` to prevent module resolution failures. This is the canonical pattern for code-bearing skills (excalidraw, video-watching). Source: src/lib/install-packs.ts; skill-packs/catalog.json.
- **config native-memory + autoMemoryEnabled:false (2026-06-05)**: dreamcontext disables Claude Code's native auto-memory on setup by default via `autoMemoryEnabled:false` in .claude/settings.json. Official key is `autoMemoryEnabled` (not memoryEnabled or native_memory). Command: `dreamcontext config native-memory enable|disable`. Module claude-settings.ts is server-safe. Source: src/cli/commands/config.ts; src/lib/claude-settings.ts.
- **Canonical vs build-output file locations (2026-06-05)**: Canonical (tracked in git): `agents/*.md`, `skill/SKILL.md`, `.codex/agents/prompts/*.md`. Build output (gitignored): `dist/` (tsup copies agents/ → dist/agents/). Local dogfood (gitignored): `.claude/`, `.agents/` — refreshed by `dreamcontext update`. When editing agent prompts or SKILL.md, update canonical locations and run `npm run build` to regenerate dist/. Mirror `.codex/agents/prompts/` manually (cp from agents/).
- **Recall engine: BM25F + stemming + synonyms + score/rank decoupling (2026-06-02)**: recall.ts uses BM25F field weighting (title×3, tags×2, description×2, body×1), TR suffix folding + EN stemming, and a query-time synonym map. Haiku relevance-rerank replaces the 8000-char positional slice. Architectural invariant: ALL new ranking signals (field weighting, recency, status, synonyms, capture penalty) feed a DERIVED `rankScore`; `hit.score` remains flat BM25. This preserves gate thresholds unchanged (BM25 fallback ≥2.0 changed to .some() in Phase-5 to survive reordering; skill gate ≥1.0; explore agent ≥5/<2). Proven on 60-query gold set: recall@1 68→85%, recall@3 82→95%. Source: src/lib/recall.ts; eval/gold.jsonl; commits a56f9d4, e31b2e3.
- **Capture guard (2026-06-02)**: auto-capture flood degraded recall (−3.3% @3, −8.3% @1). Fix: CAPTURE_RANK_PENALTY=0.5 applied to rankScore only (never hit.score), MAX_INDEXED_DIGESTS=50 hard cap. Proven 0 true displacement at N=50/100/200. Source: src/lib/session-digest.ts; commit 4585612.
- **Tauri hosting architecture (2026-05-31)**: Spawn the existing Node CLI as a child process on a free loopback port; load via WebviewUrl::External. Do NOT reimplement /api/* in Rust; do NOT serve a static SPA. Shell capability scoped to `node` only. A7 (tauri dev window) and A8 (sign/notarize/updater/icons) are always manual handoffs. Source: desktop/src-tauri/src/lib.rs; commit 6e9dfac.
- **Vault registry (2026-05-31)**: Global vault registry at ~/.dreamcontext/vaults.json. resolveVaultContextRoot() accepts a registered vault name OR an explicit path. Source: src/lib/vaults.ts.
- **catalog.ts extraction (2026-05-31)**: loadCatalog + catalog types live in src/lib/catalog.ts, not install-skill.ts. This keeps @inquirer/prompts out of the server bundle. The server imports catalog.ts directly; install-skill.ts re-exports from catalog.ts. Source: src/lib/catalog.ts; commit 2cef75b.
- **Control-plane API contextRoot convention (2026-05-31)**: Route handlers receive contextRoot = the _dream_context/ dir. Routes that need the project root MUST use dirname(contextRoot). PATCH /api/config strict-picks only {platforms, packs} — never spread the request body — with per-element validation to prevent prototype pollution. GET /api/version-check is cache-only: no network call in the request path. Source: src/server/routes/config.ts, version-check.ts; commit 2cef75b.
- **Dashboard server loopback bind + CSRF + CORS + path-traversal hardening (2026-05-31)**: dashboard server MUST bind 127.0.0.1 by default (--host opt-in). Wildcard CORS (Access-Control-Allow-Origin: *) on mutating routes was a CSRF write vector — any website open while dashboard runs could modify soul/task/sleep files. Fix: handleCors() reflects loopback origins only; Origin/Host check rejects cross-site state-changing requests; safeChildPath guards /api/core/:filename. Decision: pulled forward from v0.6 task to ship v0.5.1 clean before npm publish. Residual per-route traversal for tasks/knowledge/features tracked in v06 (already gated by CSRF + loopback). Source: src/server/index.ts, middleware.ts, safe-path.ts; commit 0bb6d7c.
- **Context gate must show full skill catalog, never pre-select** (2026-05-31, 4 iterations enforced): UserPromptSubmit skill gate must NOT list a top-3 pre-selected subset of relevant skills. The agent sees the full skill catalog already; pre-picking arbitrary top-3 constraints which skills get invoked. Gate tells agent: "review skills available to you (full list already in context) and INVOKE any that fit — do NOT limit yourself to a pre-selected few; scan them all and decide." Internal BM25 check only decides WHETHER to fire the gate, never WHICH skills to show. Source: src/cli/commands/hook.ts; commit 23c28a2.
- **Haiku single-call recall (2026-05-26)**: haikuRecall() replaces multi-query BM25 keyword extraction. A single claude CLI call (--model haiku, -p, no tools, no session persistence) receives the full corpus index (capped at 8,000 chars) embedded in a system prompt and returns 0-3 doc keys as JSON. Rationale: execFileSync over Anthropic SDK because the SDK requires ANTHROPIC_API_KEY while the claude CLI uses its own auth (the user always has it installed). Single call over multi-query because Haiku understands intent cross-language (Turkish/English mixed prompts work correctly without keyword extraction). Falls back to raw BM25 (null return) when claude CLI not available. Source: src/lib/recall-query-extractor.ts; src/lib/recall-multi-query.ts deleted.
- **multi-review coordinator removed (2026-05-26)**: coordinator sub-agent deleted from multi-review skill. Main agent synthesizes specialist reports directly. Coordinator added latency and token cost; main agent already has full diff context to dedupe/re-rank. Distinct from the original design (2026-05-24) where coordinator was the sole reader of full reports.
- **multi-review skill architecture (2026-05-24, coordinator removed 2026-05-26)**: router + 4 niche specialists; main agent synthesizes final verdict directly (see coordinator-removed entry above). Router classifies tier (Trivial/Lite/Full), emits JSON dispatch plan; specialists run in parallel scoped to their files; edge-cases specialist default-on at tier >= Lite. Every specialist declares `skills:` in YAML frontmatter (skill-aware). Distinct from `three-reviewer-parallel-mandates-pattern` (pre-implementation adversarial) and `reviewer.md` (generic single-agent).
- **Brain 3D lazy-load pattern (2026-05-23)**: Three.js/react-force-graph-3d shipped as separate lazy chunk (~334 KB gzip). React.lazy import in BrainPage.tsx means the chunk downloads only when user toggles to 3D. Main index chunk size unchanged.
- **Link-aware BM25 boost deferred (2026-05-23)**: No evidence BM25 fails on this corpus (44 docs, 94.7% top-1 verified). Decision: collect a gold-set of queries where BM25 misfires (5–10 examples) before implementing link traversal or embedding overlay. Revisit at that threshold.
- **Mermaid shared hook pattern (2026-05-23)**: dashboard/src/lib/mermaidRender.ts is the canonical integration point for mermaid in the dashboard. Do not call mermaid init or renderMermaidIn directly in components — route through useMermaidRender / initMermaid(theme). Theme comes from useTheme().resolved (data-theme), not prefers-color-scheme.
- **5→3 sleep specialist collapse (2026-05-10)** is canonical. No fallback. `sleep-tasks` + `sleep-state` always fire; `sleep-product` conditional. (CHANGELOG `2026-05-10|sleep`.)
- **rem-sleep orchestrator agent permanently removed (2026-05-09)**. Main agent fans out directly per SKILL.md. Sub-agents cannot reliably fan out to other sub-agents in Claude Code.
- **Anti-bloat cap 150 lines (2026-05-23)** per core file. Tightened from 300 after `memory recall` shipped — archived content stays findable on demand.
- **mem0 explicitly rejected (2026-05-23)** in favor of deterministic BM25. Detail: `knowledge/decision-mem0-vs-bm25-recall.md`. No fallback.
- **3-reviewer parallel-mandates pattern (2026-05-23)** for pre-implementation adversarial review. Detail: `knowledge/three-reviewer-parallel-mandates-pattern.md`.
- **Memory recall hook ON by default (2026-05-23)**. Opt-out via `DREAMCONTEXT_MEMORY_HOOK=0`. Score ≥2.0 + prompt ≥8 chars filters keep noise low.
- **Snapshot drops Sleep State + inlined Pinned Knowledge (2026-05-23)**. Hook delivers debt; pinned files surface at top of Knowledge Index with prominent warning, content via recall or Read.
- **CHANGELOG schema: summary + references + supersedes (2026-05-23, Option E)**. `summary` ≤200 char soft target for snapshot display; `references[]` flat strings with prefix convention `commit:|file:|knowledge:|feature:|task:|url:`; `supersedes` optional pointer to prior reversed entry. All three optional, backwards-compatible. Recall corpus now includes CHANGELOG entries (was: knowledge + feature + task + memory only).
- **Dashboard mermaid sanitizer (2026-05-02)**: line-based parser (not regex) auto-quotes labels with regex chars. Unescaped `]` still requires manual quoting (mermaid grammar limitation).
- **dreamcontext is npm-linked globally**: `~/.nvm/.../bin/dreamcontext` symlinks to this repo's `dist/index.js`. Every root `npm run build` auto-propagates.
- **Root `npm run build` required for dashboard changes**: `dashboard/npm run build` builds only `dashboard/dist/`. CLI serves from `dist/dashboard/`, populated by tsup `onSuccess`. Burned 3 debugging rounds.
- **Council UI**: final report = primary payload, must be above fold. Overview tab = final report as hero. Dynamic section parsing (synthesizer emits `## Verdict` + `## Appendix` reliably; other sections vary). CouncilHall grid + CouncilDetail full-page navigation, not split-pane.
- **`assertSafeSegment` + `assertWithinCouncil` pattern** for user-supplied path segments: reject traversal chars before construction; assert resolved path stays under domain root; validate at every lookup site.
- **SubagentStart outputs JSON, SessionStart outputs plain text**. Hook spec requirement.
- **Stop hook is synchronous** (`readFileSync(0)`) — guarantees `.sleep.json` written before next SessionStart.
- **Sleep epoch-based clearing**: `sleep start` records timestamp; `sleep done` only clears records from before epoch.
- **LIFO default in `insertToJsonArray`**: `position: 'top'`. CHANGELOG.json integrity depends on this.
- **Snapshot is chalk-free**. Consumed by shell hook; ANSI breaks processing.
- **ESM-only, no bundled deps**. chalk v5+ and nanoid v5+ are ESM-only; tsup externalizes all runtime deps.

- **goal-skill validation contract (2026-05-31)**: The goal-skill loop ran 3 autonomous cycles (plan→2-3 reviewers→implement→review→validate) for v0.6 slices 1-3. Validation contract used in all three: `npm run build && npm test` (vitest + build). Plan reviewer cap = 3 iterations before escalate; code reviewer cap = 3. marketing-council.test.ts is non-deterministic (pre-existing flake; tracked in deflake-marketing-council) and is NOT a valid validation failure.

## Known Issues

- **Haiku 8K index truncation (2026-06-05)**: haikuRecall() caps the corpus index at 8,000 chars; at ~188 docs this silently drops ~half the index. CHANGELOG entries load first and crowd out knowledge/feature docs. Workaround: BM25 fallback still fires; net impact is recall degraded on large corpora. Fix: raise cap or switch to a token-count-based trim; track in ecc-inspired-roadmap task.
- **knowledge_access not updated by recall path (2026-06-05)**: knowledge_access in .sleep.json is only bumped by explicit `dreamcontext knowledge touch <slug>` calls, never by the BM25 or Haiku recall path. Staleness tracking therefore misses implicit access via recall hooks. Fix: call bumpKnowledgeAccess() in recall.ts whenever a doc is returned as a top hit.
- **CLI version string hardcoded** — FIXED in v0.5.0. dreamcontextVersion() now reads package.json at runtime; .version() call updated.
- **1 pre-existing flaky test**: `tests/unit/id.test.ts` `generates 8-char suffix after prefix` (nanoid(8) intermittently returns 5 chars). 743 tests total, all passing in normal runs.
- Integration tests require manual `npm run build` before they can run against `dist/index.js`.
- `getTemplateDir()` in `init.ts` uses candidate-path approach; unusual pnpm layouts could cause silent template resolution failure.
- **Dashboard Phase 5 pending**: a11y audit, responsive layout (sidebar collapse), i18n token extraction, bundle size audit (~80KB gzipped). Phases 1-4c complete.
