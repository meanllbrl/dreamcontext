### 2026-03-01 - UserPromptSubmit hook: persistent sleep debt reminders

Added `hook user-prompt-submit` subcommand to `src/cli/commands/hook.ts`. Fires on every user message via Claude Code's UserPromptSubmit hook. Reads sleep debt from `.sleep.json` and outputs a one-line reminder when debt >= 4. Silent when debt < 4. Critical (★★★) bookmarks override the debt threshold. Read-only — no writes to state. Same thresholds as `getConsolidationDirective()` but compact single-line messages suitable for the hook context.

Rationale: SessionStart fires once per session, which agents can dismiss or forget. UserPromptSubmit fires on every user turn — when the user responds, the agent must process the reminder again before generating its next reply. This is the closest analog to "you must consolidate" being truly persistent.

`install-skill.ts` updated to register UserPromptSubmit as the 5th hook (SessionStart, Stop, SubagentStart, PreToolUse, UserPromptSubmit). `skill/SKILL.md` YAML frontmatter updated with hook declaration and Sleep System docs expanded to describe persistent reminders. 6 new integration tests. **415 tests total, all passing.**

## Technical Decisions

<!-- High-signal decisions only. Granular implementation decisions archived in knowledge/technical-decisions-archive.md -->

- **5→3 sleep specialist collapse (2026-05-10)**: Original v0.3.0 fan-out had 5 specialists (sleep-tasks, sleep-changelog, sleep-core, sleep-knowledge, sleep-features). Collapsed to 3: sleep-state (owns core 0-6 + CHANGELOG + RELEASES + triggers, always fires), sleep-product (owns knowledge/ + feature PRDs, conditional), sleep-tasks (unchanged, always fires). Rationale: parallel agents help wall-clock only to the slowest lane (sleep-tasks); merging the others reduces launch overhead + synthesis cost without slowing the floor. Merged agents: sleep-core + sleep-changelog → sleep-state; sleep-knowledge + sleep-features → sleep-product. Non-overlapping file domains preserved — no-stomp guarantee unchanged. Sub-passes A/B/C inside sleep-state prompt handle the identity-vs-diary tension from the merge. This is the canonical sleep architecture; there is no fallback path.
- **rem-sleep orchestrator agent permanently removed (2026-05-09)**: `agents/dreamcontext-rem-sleep.md` and `.codex/agents/prompts/dreamcontext-rem-sleep.md` deleted. No fallback path. Main agent fans out directly to 5 specialists per SKILL.md. Rationale: sub-agents cannot reliably fan out to other sub-agents in Claude Code (verified by failed live test). "One path. No fallback. No degradation vector."
- **rem-sleep is a fan-out system (2026-05-09)**: Monolithic rem-sleep replaced by 5 domain specialists (sleep-tasks, sleep-changelog, sleep-core, sleep-knowledge, sleep-features). Orchestration runs from the main agent context via SKILL.md step-by-step instructions. Orchestrator always dispatches sleep-tasks + sleep-changelog + sleep-core in parallel; conditionally dispatches sleep-knowledge and sleep-features based on signals. No shared digest file — specialists call CLI directly.
- **Dashboard mermaid rendering with sanitizer (2026-05-02)**: `TaskDetailPanel.tsx` renders mermaid flowcharts post-render. Sanitizer uses a line-based parser (not simple regex) to auto-quote labels containing regex chars. Pan/zoom via click-drag/scroll-wheel; fit on first render. Labels with unescaped `]` chars still require manual quoting in source — mermaid grammar limitation.
- **dreamcontext is npm-linked globally**: `~/.nvm/.../bin/dreamcontext` symlinks directly to this repo's `dist/index.js`. Every `npm run build` from the project root auto-propagates to all projects using the binary. No reinstall or re-link needed after rebuilds.
- **mk-skill: CLI is the only place that sets `ctx.dryRun = false`**: Library code accepts `ctx`, never constructs it. All mutations guard on `ctx.dryRun` and acquire the marketing lock + write a `runs/` WAL entry.
- **mk-skill: `meta-fetch.ts` is the sole Graph API wrapper**: No other file may call `graph.facebook.com`. Header-only auth at library level (`HeaderAuthAssertionError` if `access_token=` in URL). Three-layer fallback: typed client → `api-reference.md` → live Meta docs (dry-run first, write recipe back, propose typed wrapper after 3 uses).
- **mk-skill: budget never defaults**: `parseDailyBudget` throws `BudgetMissingError` on missing `--daily-budget`. Strategy Optimizer emits `null + ASK_USER_FOR_BUDGET`. Hard safety constraint, no override.
- **mk-skill: `mk launch` requires `--confirm <cohort_id>` verbatim**: 6-line summary BEFORE WAL creation. Entity flips one at a time with `noRetry: true`. `mk launch resume <run_id>` replays from WAL; rejects ctx mismatch.
- **mk-skill: learnings live under `knowledge/marketing-learnings/`, NOT `marketing/`**: Survive a `mk init` wipe. Per-day `.md` + `.index.json` sidecar; quarterly `_archive-<YYYY-Q>.md` via `mk rem-sleep`. Performance Monitor is the only writer (`--agent performance-monitor`; test override `MARKETING_LEARNINGS_AGENT_OVERRIDE=1`).
- **mk-skill: `mk rem-sleep` is a deterministic CLI verb**: dreamcontext-rem-sleep agent calls `dreamcontext mk rem-sleep` (Step 5d). Pure fns: pruneRuns/compactInsights/mergeDailyLearnings/redactRunsSweep. Idempotent: merge checks archive-marker before appending.
- **mk-skill: Reinfluence artifacts strictly under `_dream_context/marketing/`**: `mk init` copies to `.tools/`, creates `.venv/`, pins caches via env vars. System prereqs (python3 >= 3.10, ffmpeg) checked by health probe; cannot be bundled.
- **Root `npm run build` required for dashboard changes**: `dashboard/npm run build` builds only `dashboard/dist/`. CLI serves from `dist/dashboard/`, populated by tsup `onSuccess`. This burned 3 debugging rounds.
- **Council UI: final report is the primary payload, must be above fold**: Synthesized report buried in a non-default tab or behind a drawer will be rejected. Overview tab = final report as hero. Dynamic section parsing (final-report.md emits `## Verdict` + `## Appendix` reliably; other sections vary).
- **Council UI: Hall grid + Detail full-page navigation, not split-pane**: CouncilHall = searchable card grid; CouncilDetail = full-page with back button. Split-pane rejected (inefficient sizing).
- **install-claude-md is standalone, not bundled with install-skill**: CLAUDE.md is a project-root file, not a `.claude/skills/` entry. Idempotency via `<!-- dreamcontext:start/end -->` fences.
- **assertSafeSegment + assertWithinCouncil for user-supplied path segments**: Two guards: (1) reject traversal chars before path construction; (2) assert resolved path stays under domain root. Validate at every lookup site. This is the correct pattern for any CLI command taking external identifiers as file path segments.
- **SubagentStart outputs JSON, SessionStart outputs plain text**: Hook spec requirement. Never conflate.
- **Stop hook is synchronous**: `readFileSync(0, 'utf-8')`. Guarantees `.sleep.json` written before next SessionStart fires.
- **Sleep epoch-based clearing**: `sleep start` records timestamp; `sleep done` only clears records from before epoch.
- **LIFO default in insertToJsonArray**: `position: 'top'` is default. CHANGELOG.json integrity depends on this. Do not change without auditing all callers.
- **Snapshot is chalk-free**: Consumed by shell hook. ANSI codes break hook processing.
- **ESM-only, no bundled deps**: chalk v5+ and nanoid v5+ are ESM-only. tsup externalizes all runtime deps.
