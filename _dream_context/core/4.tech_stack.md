## Tech Stack

TypeScript 5.7, Node.js >= 18, ESM-only npm package. CLI + local HTTP server + React SPA dashboard.

## Architecture Overview

Single-binary CLI tool that can also serve a local web dashboard. No external services. All state lives on the local filesystem in `_dream_context/` directories within user projects.

**Source layout** (abbreviated; full tree: `ls src/` or `ls dashboard/src/`):
- `src/cli/commands/` -- one file per command: init, setup, core, features, tasks, knowledge, sleep, hook, snapshot, dashboard, config, reflect, vaults, update, upgrade, memory, install-skill, feedback, connections, federation, asset-drift (computeUsedAssetsChanged + cache refresh; v0.8.4)
- `src/server/` -- createServer(), router.ts (:param + *param rest-route for subdir slugs), middleware.ts (CORS/CSRF/loopback + X-Dreamcontext-Vault per-request vault binding), safe-path.ts, routes/ (tasks, sleep, core, knowledge, features, changelog, config, packs, version-check, vaults, health, connections, federation, launcher [GET /api/launcher/status, /federation-graph, POST /update, /unregister, /connection, /connection/remove, /shareable], sleepy [GET /api/sleepy/anim?mode= animated WebP], ui-settings [GET/POST /api/ui-settings/brain — _dream_context/state/.brain-settings.json], knowledge-assets [GET /api/knowledge-assets/:slug — resolves Obsidian ## Embedded Files wikilinks to base64 WebP, containment-guarded, 256-entry LRU cache], recall [GET /api/recall?q=&types=&top= BM25 corpus search; GET /api/recall/haiku — intent-aware Haiku-filtered recall for Ask mode], sleepy-chat [POST /api/sleepy-chat — spawn claude -p SSE stream with --permission-mode plan + --disallowedTools for read-only vault Q&A; GET /api/sleepy-chat/events/:id SSE poll; DELETE /api/sleepy-chat/session; session persistence in state/; normal/intelligent model tiers], board [GET /api/board — returns {shared, local} blobs; PUT /api/board/shared → overrides/board.json (version-controlled, team views); PUT /api/board/local → state/board.local.json (gitignored, per-machine)], agent-terminal [WS /api/agent/terminal?vault=&bypass= — node-pty bridge spawning claude via $SHELL -ilc in vault root; GET /api/agent/capabilities; POST /api/agent/open-terminal osascript fallback; desktop-only + loopback-only; bypassPermissions OFF by default])
- `src/lib/` -- context-path, frontmatter, markdown, section-insert, task-query (filter/group/sort), json-file, id, recall (BM25F+stemming+synonyms+TR morphology), recall-synonyms, session-digest, salience, recall-query-extractor (haikuRecall), data-structures-migration, diagrams-migration, knowledge-index, excalidraw-text, taxonomy, setup-drift, migration-runner + migration-ledger + wikilink-rewrite, vaults, vault-discovery (discoverVaults + discoverVaultsAsync), catalog, claude-settings, reflection, manifest, setup-config, snapshot-budget, feedback, connections, federation-config, federation-recall (live cross-vault BM25 recall — read-only, no copy; v0.8.5 pivot), federation-inbox (inert; copy-based drain parked on roadmap), federation-ingest (inert), federation-digest (inert), asset-drift-cache (state/.asset-drift.json keyed on cliVersion+setupVersion; v0.8.4)

- `src/migrations/` -- versioned migration registry (0.7.0.ts, 0.8.0.ts for federation connections/inbox/shareable init)

## Key Dependencies

**CLI runtime** (all externalized from bundle):
- `commander@^13.1.0` — CLI argument parsing and subcommand structure
- `chalk@^5.4.1` — Terminal colors. v5+ is ESM-only.
- `gray-matter@^4.0.3` — YAML frontmatter parsing and serialization.
- `@inquirer/prompts@^7.10.1` — Interactive prompts.
- `fast-glob@^3.3.3` — File system glob matching.
- `nanoid@^5.1.2` — Unique ID generation.

**Dashboard runtime** (separate `dashboard/package.json`, not in CLI bundle):
- `react@^19` + `react-dom@^19` — UI framework
- `@tanstack/react-query@^5` — Server state management
- `vite@^6` + `@vitejs/plugin-react@^4` — Build tool and dev server
- `react-force-graph-2d@^1.29.1` — 2D Brain graph canvas (existing)
- `react-force-graph-3d@^1.29.1` + `three@^0.184.0` — 3D Brain graph (lazy-loaded chunk, ~334 KB gzip; only downloaded when user switches to 3D view)
- `@excalidraw/excalidraw@^0.18.0` — Excalidraw board rendering in KnowledgePage; lazy-loaded via ExcalidrawPreview (only when an .excalidraw.md file is opened)
- `mermaid@^11.14.0` — Flowchart rendering in MarkdownPreview and TaskDetailPanel (shared useMermaidRender hook in dashboard/src/lib/mermaidRender.ts)
- `marked@^15.0.0` + `panzoom@^9.4.4` — Markdown rendering and pan/zoom in dashboard
- `@tauri-apps/plugin-global-shortcut` — Sleepy: register/unregister global hotkeys from the launcher window (desktop-only; not in npm package)
- `@xterm/xterm@^5.5.0` + `@xterm/addon-fit` + `@xterm/addon-webgl` + `@xterm/addon-web-links` — Agent terminal (dashboard deps): xterm.js core + GPU WebGL renderer (canvas fallback on context-loss) + FitAddon + link detection
- `node-pty@^1.1.0` — PTY spawn for Agent terminal WS bridge (root package, optionalDependency + external; dynamic import with graceful degrade; ensurePtyHelperExecutable() chmods prebuilt spawn-helper at runtime; desktop-only)
- `ws@^8.21.0` — WebSocket server for Agent terminal WS upgrade route (root package, bundled via tsup noExternal)

**Dev**:
- `tsup@^8.4.0` — Bundles CLI + server to `dist/index.js`. Copies `dashboard/dist/` to `dist/dashboard/` via `onSuccess`.
- `typescript@^5.7.3` — Strict mode.
- `vitest@^3.0.6` — Test runner (unit + integration).
- `@playwright/test` — Browser e2e specs in `e2e/` against the served dashboard.
- `sharp` — Image processing (build/docs tooling + optional runtime: GET /api/knowledge-assets/:slug uses sharp to downscale+recompress board embedded images to WebP; falls back to originals when sharp is unavailable).
- `@types/node@^22.13.4` — Node.js type definitions.

## Build Pipeline```
npm run build
  └─ build:dashboard   →  vite build in dashboard/ → dashboard/dist/
  └─ build:cli         →  tsup src/cli/index.ts + src/server/ → dist/index.js (self-contained; noExternal bundles all 7 runtime deps + createRequire shim)
                           onSuccess: cp dashboard/dist/ → dist/dashboard/```## Infrastructure

- **Distribution**: npm public registry (`dreamcontext`). Install: `npm install -g dreamcontext` or `curl | sh` (install.sh in package; v0.5.0). Manual publish via `npm publish`.
- **Build output**: `dist/` (ESM `dist/index.js` + `dist/templates/` + `dist/agents/` + `dist/dashboard/`).
- **Dashboard**: Served locally on port 4173 (configurable, --host for LAN opt-in). Not daemonized.
- **Testing**: vitest. Integration tests run against compiled binary. 1975 tests passing (post v0.9.0 multi-assignee + federation-consent/purge tests). 1 pre-existing flake (marketing-council.test.ts, non-deterministic); recall-eval gold-set drift also pre-existing. Eval harness at eval/ (gold.jsonl, gold-heldout.jsonl, harness.ts, BASELINE.md, RESULTS.md — not in npm package). Recall eval: scripts/recall-ab.ts + scripts/recall-diag.ts.
- **CI/CD**: `.github/workflows/desktop-release.yml` — on v* tag push: macos-14 (aarch64) Tauri .app build → ad-hoc deep-sign with entitlements → dreamcontext-beta_<ver>_aarch64.app.tar.gz + .sha256 → non-draft GitHub Release. install.sh post-CLI installs desktop app on macOS (best-effort, DREAMCONTEXT_INSTALL_NO_APP opt-out).
