## Tech Stack

TypeScript 5.7, Node.js >= 18, ESM-only npm package. CLI + local HTTP server + React SPA dashboard + Lab analytics-insights subsystem (v0.10.7+).

## Architecture Overview

Single-binary CLI tool that can also serve a local web dashboard. No external services. All state lives on the local filesystem in `_dream_context/` directories within user projects.

  └─ build:dashboard   →  vite build in dashboard/ → dashboard/dist/
  └─ build:cli         →  tsup src/cli/index.ts + src/server/ → dist/index.js (self-contained; noExternal bundles all 7 runtime deps + createRequire shim)
                           onSuccess: cp dashboard/dist/ → dist/dashboard/```## Infrastructure

- **Distribution**: npm public registry (`dreamcontext`). Install: `npm install -g dreamcontext` or `curl | sh` (install.sh in package; v0.5.0). Manual publish via `npm publish`.
- **Build output**: `dist/` (ESM `dist/index.js` + `dist/templates/` + `dist/agents/` + `dist/dashboard/`).
- **Dashboard**: Served locally on port 4173 (configurable, --host for LAN opt-in). Not daemonized.
- **Testing**: vitest (forks pool + concurrency 2 on CI — thread pool starved worker→main RPC on contended runners, f4ee5a3). Integration tests run against compiled binary; CI builds the CLI before tests (d4e9fee). 2838 tests passing (as of the Lab adversarial-review pass, 2026-07-06). Eval harness at eval/ (gold.jsonl, gold-heldout.jsonl, harness.ts, BASELINE.md, RESULTS.md — not in npm package). Recall eval: scripts/recall-ab.ts + scripts/recall-diag.ts.
- **CI/CD**: `.github/workflows/ci.yml` (added v0.10.5 prep, NEW — the project's first CI gate) — runs `npm test` on every push to main and every PR, Node 20 runner. `.github/workflows/desktop-release.yml` — on v* tag push: macos-14 (aarch64) Tauri .app build → ad-hoc deep-sign with entitlements → dreamcontext-beta_<ver>_aarch64.app.tar.gz + .sha256 → non-draft GitHub Release; the four pinned actions were bumped off Node 20 onto Node 24-native majors (checkout@v7, setup-node@v6, cache@v6, action-gh-release@v3). install.sh post-CLI installs desktop app on macOS (best-effort, DREAMCONTEXT_INSTALL_NO_APP opt-out).
