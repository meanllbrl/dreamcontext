## Tech Stack

TypeScript 5.7, Node.js >= 18, ESM-only npm package. CLI + local HTTP server + React SPA dashboard.

## Architecture Overview

Single-binary CLI tool that can also serve a local web dashboard. No external services. All state lives on the local filesystem in `_dream_context/` directories within user projects.

**Source layout** (abbreviated; full tree: `ls src/` or `ls dashboard/src/`):
- `src/cli/commands/` -- one file per command: init, setup, core, features, tasks, knowledge, sleep, hook, snapshot, dashboard, config, reflect, vaults, update, upgrade, memory, install-skill, feedback
- `src/server/` -- createServer(), router.ts (:param + *param rest-route for subdir slugs), middleware.ts (CORS/CSRF/loopback), safe-path.ts, routes/ (tasks, sleep, core, knowledge, features, changelog, config, packs, version-check, vaults, health)
- `src/lib/` -- context-path, frontmatter, markdown, section-insert, task-query (filter/group/sort), json-file, id, recall (BM25F+stemming+synonyms+TR morphology), recall-synonyms, session-digest (partial-digest support for PreCompact), salience, recall-query-extractor (haikuRecall), data-structures-migration, diagrams-migration, knowledge-index (recurses subdirs for subdir-qualified slugs), excalidraw-text (extractExcalidrawText: text-only indexing of boards), taxonomy (faceted vocabulary, core/taxonomy.json), setup-drift (setupVersion drift directive), migration-runner + migration-ledger (state/.migrations.json) + wikilink-rewrite, vaults, catalog, claude-settings, reflection, manifest, setup-config, snapshot-budget (token budget ladder), feedback (upstream issue filing)
- `src/lib/task-backend/` -- provider-agnostic TaskBackend: types, local.ts (file backend + normalizeBacklogFields), clickup.ts + clickup-map/clickup-fields, api-adapter, merge.ts (two-way delta merge + conflicts + WAL), identity, secrets (token storage), sync-state, git-hooks; conformance suite pins behavior across backends
- `src/migrations/` -- versioned migration registry, one module per version transition (e.g. 0.7.0.ts)
- `dashboard/src/` -- React 19 SPA; pages/ (incl. TaxonomyPage), components/ (EisenhowerMatrix+eisenhower.ts, MarkdownPreview+codeHighlight/highlight.js, KanbanBoard, FlowDiagram, About sections, layout/FullscreenOverlay for full-screen knowledge viewing), lib/ (sql-parser, mermaidRender, codeHighlight), hooks/ (TanStack Query), contexts/
- `skill/SKILL.md`, `agents/` (6 agents: initializer, explore, sleep-tasks, sleep-state, sleep-product, sleep-migration), `skill-packs/` (catalog.json; engineering, design, growth, brand-voice packs; system-prompts, excalidraw, video-watching standalone; 6 optional agents)
- `e2e/` -- Playwright specs against the dashboard (alignment, control-panel, knowledge-fullscreen)
- `desktop/src-tauri/` -- Tauri 2.x shell scaffold; spawns Node CLI on loopback, loads via WebviewUrl::External; window launch is manual

## Key Dependencies

**CLI runtime** (all externalized from bundle):
- `commander@^13.1.0` — CLI argument parsing and subcommand structure
- `chalk@^5.4.1` — Terminal colors. v5+ is ESM-only.
- `gray-matter@^4.0.3` — YAML frontmatter parsing and serialization.
- `@inquirer/prompts@^7.10.1` — Interactive prompts.
- `fast-glob@^3.3.3` — File system glob matching.
- `nanoid@^5.1.2` — Unique ID generation.

**Dashboard runtime** (separate `dashboard/package.json`, not in CLI bundle):
- `react@^19` + `react-dom@^19` — UI framework
- `@tanstack/react-query@^5` — Server state management
- `vite@^6` + `@vitejs/plugin-react@^4` — Build tool and dev server
- `react-force-graph-2d@^1.29.1` — 2D Brain graph canvas (existing)
- `react-force-graph-3d@^1.29.1` + `three@^0.184.0` — 3D Brain graph (lazy-loaded chunk, ~334 KB gzip; only downloaded when user switches to 3D view)
- `@excalidraw/excalidraw@^0.18.0` — Excalidraw board rendering in KnowledgePage; lazy-loaded via ExcalidrawPreview (only when an .excalidraw.md file is opened)
- `mermaid@^11.14.0` — Flowchart rendering in MarkdownPreview and TaskDetailPanel (shared useMermaidRender hook in dashboard/src/lib/mermaidRender.ts)
- `marked@^15.0.0` + `panzoom@^9.4.4` — Markdown rendering and pan/zoom in dashboard

**Dev**:
- `tsup@^8.4.0` — Bundles CLI + server to `dist/index.js`. Copies `dashboard/dist/` to `dist/dashboard/` via `onSuccess`.
- `typescript@^5.7.3` — Strict mode.
- `vitest@^3.0.6` — Test runner (unit + integration).
- `@playwright/test` — Browser e2e specs in `e2e/` against the served dashboard.
- `sharp` — Image processing (build/docs tooling).
- `@types/node@^22.13.4` — Node.js type definitions.

## Build Pipeline

```
npm run build
  └─ build:dashboard   →  vite build in dashboard/ → dashboard/dist/
  └─ build:cli         →  tsup src/cli/index.ts + src/server/ → dist/index.js
                           onSuccess: cp dashboard/dist/ → dist/dashboard/
```

## Infrastructure

- **Distribution**: npm public registry (`dreamcontext`). Install: `npm install -g dreamcontext` or `curl | sh` (install.sh in package; v0.5.0). Manual publish via `npm publish`.
- **Build output**: `dist/` (ESM `dist/index.js` + `dist/templates/` + `dist/agents/` + `dist/dashboard/`).
- **Dashboard**: Served locally on port 4173 (configurable, --host for LAN opt-in). Not daemonized.
- **Testing**: vitest. Integration tests run against compiled binary. 1327 tests passing (72 todo stubs), 2 skipped (98 test files). 1 pre-existing flake (marketing-council.test.ts, non-deterministic). Eval harness at eval/ (gold.jsonl, gold-heldout.jsonl, harness.ts, BASELINE.md, RESULTS.md — not in npm package). Recall eval: scripts/recall-ab.ts + scripts/recall-diag.ts.
